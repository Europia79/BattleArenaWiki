Victory Conditions

Changing how a match is won (victoryCondition)

Victory Conditions are the conditions under which a match will end. Currently these are the valid victory conditions

  • /<game type> setOption victoryCondition <victory condition>
  • LastManStanding: Match will end when there is only one team left (or player if teamSize: 1)
  • MobKills: Player with highest number of MobKills wins
  • PlayerKills: Player with highest number of PlayerKills wins
  • AllKills: Player with highest number of PlayerKills + MobKills wins
  • NoTeamsLeft: Match will end when all teams have exited (for single player/team games)
  • Custom: Let the plugin decide (used for games like CaptureTheFlag where the flag captures are dealt with by the plugin)

Example

  • /<game type> setOption victoryCondition PlayerKills

Team Options

Giving Teams Wool Heads

This will give players "wool" heads if there are more than 1 player on the team.

  • /<game type> setOption defaults woolTeams

This will give players "wool" heads always

  • /<game type> setOption defaults alwaysWoolTeams

Changing the number of teams

  • /<game type> setOption nTeams <range>

Example 1

only 2 teams will play

  • /<game type> setOption nTeams 2

Example 2

Allow 3 teams or up to 5 teams play

  • /<game type> setOption nTeams 3-5

Example 3

allow 2 or more teams to play

  • /<game type> setOption nTeams 2+

Changing the number of players on each team

  • /<game type> setOption teamSize <range>

Example 1

Example to set 4 players per team:

  • /<game type> setOption teamSize 4

Example 2

Example to set 2 through 5 players per team:

  • /<game type> setOption teamSize 2-5

Example 3

Allow any number of players to join a team

  • /<game type> setOption teamSize 1+

Match & Event Options

Changing the number of lives

  • /<game type> setOption nLives <lives or infinite>

Examples

  • /<game type> setOption nLives 3
  • /<game type> setOption nLives infinite

Changing the length of a match

  • /<game type> setOption matchTime <seconds>

Example

Set the game to last 4 minutes. aka 4 * 60 = 240

  • /<game type> setOption matchTime 240

Infinite Time Example

  • /<game type> setOption matchTime infinite


Changing the force start time

  • /<game type> setOption forceStartTime <seconds>

Example

Set the forcestart time to 15 second (default is 60)

  • /<game type> setOption forceStartTime 15

Giving money for winning/losing

  • /<game type> setOption winner money <amount>
  • /<game type> setOption losers money <amount>

Giving items for winning/losing

Currently has to be set manually. [items]

Giving money for getting first place/participating in a tournament

  • /<game type> setOption firstPlace money <amount>
  • /<game type> setOption participants money <amount>

Giving items for getting first place/ participating in a tournament

Currently has to be set manually. [items]

Clearing Inventory on Death

  • /<game type> setOption onDeath clearInventory

Respawning on Death

You need to give the players more lives.

  • /<game type> setOption nLives <amount>

Giving players random respawn locations

If you want players to respawn at random spawn points (You should probably create a bunch of spawn locations )

  • /<game type> setOption onDeath randomRespawn

Examples

  • /<game type> setOption nLives 3
  • /<game type> setOption nLives infinite

Respawning with class items

  • /<game type> setOption onSpawn respawnWithClass

Running commands

You can run commands at any transition, onSpawn, onStart, winners, etc.

  • /<game> setOption <transition> commands <command to run>

The format for the command is

  • <player or console> <commands that you want to run>

The first player or console makes you run as the command as the operating console or as the player.

If in the commands the word "player" appears it will then be substituted with the player name.

For example. Lets say alkarin_v is entering the arena, and the command is "- player fly stop player". This will run the command "fly stop alkarin_v" as the player "alkarin_v"


Example

Running a command for everyone when they spawn in a skirmish. This will run from the console the command "Hello <the player name>". So if lDucks entered it would say "Hello lDucks" every time they spawn

  • /skirmish setOption onSpawn commands console Hello player

Giving default classes

Can be given at any stage, but makes the most sense whenever they teleport in(usually onJoin or onStart), or onSpawn.

  • /<game> setOption <stage> giveClass <classname>

If you want people to respawn with their given class

  • /<game> setOption onSpawn respawnWithClass

Example

Giving people in skirmish the gladiator class onStart

  • /skirmish setOption onSpawn giveClass gladiator


Giving enchants

Can be given at any stage, but makes the most sense whenever they teleport in(usually onJoin or onStart), or onSpawn. Enchants can also be added to classes. For the correct names see Enchants.

  • /<game> setOption <stage> enchants <enchant name:power:length>

Example

Giving people in skirmish speed:3 for 90 seconds onStart

  • /skirmish setOption onSpawn enchants speed:3:90


Giving Disguises

Currently must be done manually. See Giving disguises


Auto rejoining after a game

  • /<game> setOption onComplete rejoin



Making the Game Always Joinable

For this to work the game should have a variable number of teams, or teamsize. For example 2-6 teams with teamSize 1, or 2 teams with teamSize 1+. Afterwards you use this command

  • /<game> setOption onJoin alwaysJoin

If you have a waitroom with the game by default waitRooms close while a game is running, so you will also have to specify the command to not close these.

  • /<game> setOption waitroomClosedWhileRunning false

Making the Game Always Open

Always open games should have an infinite time.

  • /<game> setOption matchTime infinity

Now set the default option to make the game always open (You should probably also make the game always joinable)

  • /<game> setOption defaults alwaysOpen

Joining Options

Limit arena joining to same world

  • /<game type> setOption preReqs sameWorld


Limit arena joining to within a certain distance of an arena

Distance is based off of distance to spawn points and is the number of blocks from the spawn points.

  • /<game type> setOption preReqs withinDistance <distance>

Allow Players to join at any time

  • /<game type> setOption onJoin alwaysJoin

Adding a price to join

  • /<game type> setOption preReqs <money>

Stages and Transitions

All stages and transitions can have any of the 'options'.

Match Stages

A match is composed of different stages.

  1. onOpen: For events this happens when you open the event. For matches when players get matched from the queue.
  2. onPrestart: time between onOpen and onStart.
  3. onStart: Match starts
  4. onVictory: One player or team is victorious
  5. onComplete: match was completed successfully
  6. onCancel: match was cancelled
  7. onFinish: happens after onComplete or onCancel

Match Transitions

During the match many different transitions can happen

  1. onSpawn: Happens on teleport or on player Respawn ( if respawn is set within onDeath)
  2. onDeath: What happens when a player dies
  3. onJoin: player joining a match
  4. onEnter: Happens when the player first enters the arena. right after before they are teleported in.
  5. onLeave: Happens when the player leaves the arena(aka if they win(after onComplete), or if they die(and dont respawn) )
  6. losers: Happens after onVictory, before onComplete, used for prizes
  7. winner: Happens after onVictory, before onComplete, used for prizes
  8. drawers: Happens after onVictory, before onComplete, used for prizes
  9. firstPlace: Prizes for tournaments
  10. participants: Prizes for tournaments

Changing options per Arena

You can set different options per arena if you would like. All of the commands are available you just use a different command. The command is

  • /<game type> setArenaOption <arena name> whateverOptionYouAreSetting [maybe some arguments if the option needed it]

Example

As an example say you are trying to change the number of teams for an arena called IcyDeath to be 4 teams in the game type skirmish

  • /skirmish setArenaOption IcyDeath nTeams 4
