config.yml

This is the main config that will affect options globally for BattleArena and all games and extension games (unless you set up those games to override these defaults). This config is found at BattleArena/config.yml inside of your plugins folder. If you mess something up you can revert to the current default [config.yml].

Reloading the config

Once you change a config option you will need to reload the config.

  • /arena reload

Or if you are changing the config of one of the plugins that uses BattleArena(like CTF, Spleef, or Paintball), reload the appropriate Config. Spleef Example:

  • /spleef reload

If you want to know how to change individual games

Changing the Game configs : like SkirmishConfig.yml

Options

Option Values Description
autoUpdate true or false Updates will be retrieved from the latest plugin on the bukkit site
moneyName String What is the money on this server called, or if you use Vault the money name will come from there
teleportYOffset float How many blocks up or down should be adjusted when teleporting players (mainly used for servers where lag causes players to sink a bit when teleporting)
teleportYVelocity float Change the player velocity after being teleported
numberSavedInventories Integer how many old inventories should be saved for each player. 0 for no saving. Stored Inventories can be given to players using the /ba commands
disabledCommands List of String which player commands should be disabled when they enter an arena
disabledQueueCommands List of String which player commands should be disabled when they are in the queue
disabledHeroesSkills List of commands which heroes skills should be disabled when they enter an arena
ignoreMaxStackSize True or False If set to true, items that are usually not stackable will be stacked when a player is given items. Examples: 64 mushroom soup, or 64 snow_ball, instead of 1 or 16
unsafeItemEnchants True or False if set to true, given enchanted items will not be limited to 'normal' safe enchantment levels. Example, most weapons are limited to sharpness 5. if unsafeItemEnchants: true, this can be any level
useArenasOnlyInOrder True or False if set to true, arenas will cycle in order. Example: If true if a player joins a match which has 2 arenas. 1v1 and 1v1v1v1. Then 1v1 will happen first afterwards the 1v1v1v1 is guaranteed to be the next arena used.
enableInvisibleTeleportFix True or False Bukkit or Minecraft had a bug that sometimes caused players to be invisible after teleporting. If this is happening on your server set this to true. This option will be taken away once I have confirmed bukkit has fixed the problem

defaultOptions

These options are the default for Game Types. Changing the defaultOptions: will change it for all games unless the Game Type specifically overrides.

Time Options

Option Values Description
secondsTillBegin Integer UNUSED AT THIS TIME
secondsTillMatch Integer how long between onPrestart and onStart. This is the time that players will have to choose classes if there is a waitroom for the game.
matchTime Integer or "infinite" ## How long do timed matches last, (in seconds). put "infinite" for a match that never times out
secondsToLoot Integer Time after winning that players have before being teleported out of the arena
matchUpdateInterval Integer For timed matched, how long between sending players match updates
matchForceStartTime Integer When games have a variable amount of players, how long should people be allowed to join before it automatically starts.

Message Options

These are the message options, they only affect the broadcasts to the server or channel, not the messages the fighting players receive. They are found in the section defaultOptions.announcements or defaultOptions.eventAnnouncements(used for tournaments). Values are one of the following, announce or dontannounce. If you specify announce by default that will broadcast to the server, just like specifying server or you can specify a channel for chat plugins (this needs [Vault]). Message Options

  • announce : announce these messages
  • dontannounce : dont announce these messages
  • server : broadcast to the server
  • channel=<channel> : use the channel specified (needs the Vault plugin)
Option Values Description
onCountDownToEvent Options match is now open
onPreStart Options match going to happen soon, example 'P1[p1Elo] vs P2[p2elo]'
onStart Options match starting
onVictory Options match has been won, exmaple 'P1[p1elo] has defeated P2[p2elo]'
onEnterQueue Options player1 has entered the queue for <match type>

Example

Announce the upcoming match that will happen soon to the server, like 'Player1 vs Player2'

  • onPreStart: [ announce, server ]

The same as above but announcing it to the chat channel 'PvP' (needs [Vault])

  • onPreStart: [ announce, channel=PvP ]

Or if you don't want an announcement

  • onPreStart: [ dontannounce ]
