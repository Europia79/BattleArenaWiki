Adding regions with WorldGuard

Adding a worldGuard region to your arena

First, use your //wand to select an area, then type the command

  • /<game> alter <arena name> addregion


Valid Options

More options will be added as requested

  • wgNoEnter : players cant enter the arena region (except for fighting players)
  • wgNoLeave : players in the arena can't leave the region
  • wgClearRegion : clears the region of any items
  • wgResetRegion : resets the region to the way it was when you added it via /arena alter addRegion.

Stop players from entering

The wgNoEnter command will make it so that no players(besides those fighting) can enter the region selected. This means the arena could be a small area in an open meadow, but only those fighting would be able to enter.

  • /<game> alter <arena name> defaults wgNoEnter

Stop players from leaving

The wgNoLeave command will make it so that the fighting players can't leave the region selected. This means the arena could be a small area in an open meadow, but those fighting would be not be able to leave the small area.

  • /<game> alter <arena name> defaults wgNoLeave

Clearing the Region

To clear the region it is best to do it when the game is finished, or when it's first opening. You can do both if you would like

  • /<game> alter <arena name> onOpen wgClearRegion
  • /<game> alter <arena name> onFinish wgClearRegion

Reseting the Region

Resetting the region should be done when the game is finished. It is important to note this can be a very time consuming command. When you type the command addRegion it makes a copy of the currently selected region. When wgResetRegion happens it will "paste" that region back. The copy and paste are both done using WorldEdit, so the time involved is exactly as much time it is to use WorldEdit to perform those commands on the region you have selected.

  • /<game> alter <arena name> onFinish wgResetRegion
